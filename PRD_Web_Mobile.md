# Product Requirements Document: PBM Knowledge Hub (Web & Mobile)

**Working Title:** PBM Knowledge Hub
**Author:** Manus AI
**Version:** 2.0
**Date:** November 11, 2025

---

## 1. Introduction

This document outlines the revised product requirements for the **PBM Knowledge Hub**, a cross-platform application (Web, iOS, Android) that serves as a definitive educational resource on photobiomodulation and integrates practical HRV biofeedback tools.

### 1.1. Project Vision

To create a high-value, comprehensive platform that combines deep educational content with practical wellness tools, establishing Neurofeedback Luxembourg as a leading authority in PBM and providing a powerful tool for client engagement and lead generation.

---

## 2. Core Features

| ID | Feature | Priority | Description |
| :-- | :--- | :--- | :--- |
| F01 | **Interactive Content Hub** | Must-have | A searchable, categorized library of all PBM and Vielight content, with interactive diagrams and embedded videos. |
| F02 | **HRV & Symptom Tracking** | Must-have | Integration of the "Coherence 365" HRV biofeedback feature and a daily symptom/mood journal. |
| F03 | **Personalized Learning Paths** | Should-have | Guided learning paths for different user personas (e.g., "New to PBM," "Practitioner," "Advanced User"). |
| F04 | **Community Forum** | Should-have | A simple, moderated forum for users to ask questions and share experiences. |
| F05 | **Lead Generation Form** | Must-have | A contact form for users to request a consultation with a Neurofeedback Luxembourg practitioner. |

---

## 3. Technical Specifications

- **Platforms:** Web, iOS, Android
- **Web Framework:** Next.js (React)
- **Mobile Framework:** React Native
- **Backend:** A Node.js backend on elest.io for managing user accounts, forum data, and lead generation forms.
- **Database:** PostgreSQL for user data and community content.

---

## 4. Obvious Improvements Incorporated

1.  **Interactive Content:** The educational content will be enhanced with interactive diagrams and quizzes to improve engagement and learning.
2.  **Personalized Experience:** Learning paths will guide users through the content based on their interests and knowledge level, making the app more valuable and less overwhelming.
3.  **Community Building:** A community forum will foster engagement, provide social proof, and create a valuable source of user-generated content and feedback.
4.  **Direct Lead Generation:** The inclusion of a clear call-to-action and contact form turns the educational app into a direct lead generation tool for the clinic.
